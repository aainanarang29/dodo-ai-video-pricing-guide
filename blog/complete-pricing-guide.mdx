# The Complete Guide to AI Video Pricing: 7 Models That Actually Work

The AI video generation market is exploding. From Runway's cinematic Gen-4 model to Pika's social-first approach, tools that once cost $100,000 in traditional production now deliver professional results for $10/month.

But here's the challenge: **the pricing models that work for traditional SaaS completely break with AI video.**

Why? Because your costs are nonlinear, GPU-intensive, and wildly unpredictable. A single 5-second video generation can cost you $0.50 in compute. Scale that to 10,000 users, and you're looking at infrastructure costs that can sink your startup overnight.

This guide breaks down the **7 pricing models** that actually work for AI video tools—backed by real data from companies processing millions in revenue, conversion rates from industry leaders, and implementation code using Dodo Payments.

## 1. Credit-Based Pricing

**Best for:** Generation-focused tools (Runway, Pika, Luma)  
**Conversion rate:** 8-15%  
**Example ARR:** $50-150 per user

### How It Works

Users purchase credit packages. Each video generation consumes credits based on:
- **Duration** (5s vs 10s)
- **Quality** (480p vs 4K)
- **Model** (Turbo vs Pro)
- **Features** (camera control, style transfer)

### Real-World Example: Runway ML

**Pricing structure:**
```
Free:     125 credits/month
Standard: $15/month → 625 credits
Pro:      $35/month → 2,250 credits
Unlimited: $95/month → Relaxed mode after credits
```

**Credit consumption:**
- Gen-4 Turbo (5s): 5 credits = ~$0.12
- Gen-4 Standard (5s): 10 credits = ~$0.24  
- Gen-4 4K upscale: +20 credits

**Why it works:**
- Aligns pricing with GPU costs
- Users understand "credits = generations"
- Natural upsell path (run out → upgrade)
- Prevents abuse with metered limits

```typescript Credit Consumption
// Deduct credits for video generation
async function generateVideo(userId: string, params: VideoParams) {
  // Calculate credit cost
  const creditCost = calculateCreditCost({
    duration: params.duration,
    quality: params.quality,
    model: params.model,
  });

  // Check balance
  const user = await db.user.findUnique({ where: { id: userId } });
  if (user.credits < creditCost) {
    throw new Error('Insufficient credits');
  }

  // Deduct credits atomically
  await db.$transaction([
    db.user.update({
      where: { id: userId },
      data: { credits: { decrement: creditCost } }
    }),
    db.creditTransaction.create({
      data: {
        userId,
        amount: -creditCost,
        type: 'video_generation',
        metadata: params,
      }
    })
  ]);

  // Generate video (queue to background job)
  return await queueVideoGeneration(userId, params);
}

function calculateCreditCost(params: VideoParams): number {
  const baseCredits = {
    '5s': 10,
    '10s': 20,
  }[params.duration];

  const qualityMultiplier = {
    '480p': 1.0,
    '720p': 1.5,
    '1080p': 2.0,
    '4K': 3.0,
  }[params.quality];

  const modelMultiplier = {
    'turbo': 0.5,
    'standard': 1.0,
    'pro': 1.5,
  }[params.model];

  return Math.ceil(baseCredits * qualityMultiplier * modelMultiplier);
}
```

## 2. Subscription Tiers with Usage Limits

**Best for:** Avatar/editing tools (Synthesia, HeyGen, CapCut)  
**Conversion rate:** 5-8%  
**Example ARR:** $200-600 per user

### How It Works

Monthly subscription with clear usage limits:
- **Minutes of video** generated/month
- **Number of exports** per month
- **Storage limits** for projects
- **Feature access** (HD vs 4K, avatar library)

### Real-World Example: Synthesia

**Pricing structure:**
```
Starter:  $29/month → 10 minutes of avatar video
Creator:  $89/month → 30 minutes + custom avatars
Enterprise: Custom → Unlimited + API access
```

**What happens at limit:**
- **Soft limit:** Generate, but can't export (paywall)
- **Hard limit:** Cannot generate more (upgrade required)
- **Overage:** Additional minutes at $9/minute

## 3. Freemium with Watermark Removal

### How It Works

Free tier allows full editing capabilities but adds watermarks to exports. Users pay one-time or subscribe to remove watermarks.

**Best for:** Editing/creation tools where users invest significant time before exporting.

### Real-World Example: VEED.io

**Pricing tiers:**

| Plan | Price | Features | Limits |
|------|-------|----------|--------|
| Free | $0 | Edit, add watermark | VEED logo on exports |
| Lite | $9/month | Remove watermark | 10min/month exports |
| Pro | $24/month | All features | 2hr/month exports |

**Why it works:**
- Sunk cost fallacy - users invest time editing before paywall
- Clear value prop - "remove logo" vs abstract features
- Low friction - one-time payment option available

  ```typescript fair-use-enforcement.ts
  import { getSubscription, getMonthlyUsage } from '@/lib/subscriptions';
  
  export async function checkFairUseCompliance(
    userId: string
  ): Promise<{
    canGenerate: boolean;
    usage: number;
    softCap: number;
    status: 'normal' | 'warning' | 'throttled';
  }> {
    const subscription = await getSubscription(userId);
    const usage = await getMonthlyUsage(userId);
    
    const softCap = parseInt(subscription.metadata.fair_use_cap);
    const warningThreshold = softCap * 0.8; // 800 generations
    const throttleThreshold = softCap * 1.5; // 1,500 generations (allow 50% overage)
    
    // Status determination
    let status: 'normal' | 'warning' | 'throttled' = 'normal';
    if (usage.generations >= throttleThreshold) {
      status = 'throttled';
    } else if (usage.generations >= warningThreshold) {
      status = 'warning';
    }
    
    // Throttling logic
    const canGenerate = usage.generations < throttleThreshold;
    
    return {
      canGenerate,
      usage: usage.generations,
      softCap,
      status,
    };
  }
  
  // Warning email at 80% usage
  export async function sendFairUseWarning(userId: string) {
    const { usage, softCap } = await checkFairUseCompliance(userId);

        to: userId,
        subject: 'Approaching fair use limit',
        body: `You've used ${usage} of ~${softCap} included generations this month.

      });
  ```

  ```typescript quality-tiers.ts
  ): Promise<number> {
    // Different quality tiers consume different GPU resources
    const costMultipliers = {
      standard: 1.0, // Baseline
      extended: 2.5, // 2.5x more compute for longer/higher quality
    };

  // Apply soft throttling for heavy users on extended mode
  export async function applySmartThrottling(
    userId: string,
    qualityTier: 'standard' | 'extended'
  ): Promise<{ allowed: boolean; queuePosition?: number }> {
    const { status } = await checkFairUseCompliance(userId);
    
    // Normal users: no throttling
    if (status === 'normal') {
      return { allowed: true };
    }
    
    // Warning users: allow but deprioritize extended mode
    if (status === 'warning' && qualityTier === 'extended') {
      return {
        allowed: true,
        queuePosition: await getQueuePosition('low_priority'),
      };
    }
    
    // Throttled users: block extended mode, queue standard
    if (status === 'throttled') {
      if (qualityTier === 'extended') {
        return { allowed: false };
      }
      return {
        allowed: true,
        queuePosition: await getQueuePosition('very_low_priority'),
      };
    }
    
    return { allowed: true };
  ```

  ```typescript credit-management.ts
  import { getSubscription } from '@/lib/subscriptions';
  
  interface CreditBalance {
    subscription_credits: number;
    purchased_credits: number;
    total: number;
    rollover_cap: number;
  }
  
  export async function getCreditBalance(
    userId: string
  ): Promise<CreditBalance> {
    const subscription = await getSubscription(userId);
    const purchases = await getUserCreditPurchases(userId);
    
    const monthlyAllowance = parseInt(subscription.metadata.monthly_credits);
    const usedThisMonth = await getMonthlyUsage(userId);
    const rolloverCap = parseInt(subscription.metadata.max_rollover);
    
    // Calculate rollover from last month
    const lastMonthUnused = await getLastMonthUnusedCredits(userId);
    const rolledOver = Math.min(lastMonthUnused, rolloverCap);
    
    // Subscription credits = monthly allowance + rolled over
    const subscriptionCredits = monthlyAllowance + rolledOver - usedThisMonth.credits;
    
    // Purchased credits never expire
    const purchasedCredits = purchases.reduce(
      (sum, p) => sum + parseInt(p.metadata.credits),
      0
    );
    
    return {
      subscription_credits: Math.max(0, subscriptionCredits),
      purchased_credits,
      total: Math.max(0, subscriptionCredits) + purchasedCredits,
      rollover_cap,
    };
  }
  
  export async function deductCredits(
    userId: string,
    amount: number
  ): Promise<{ success: boolean; new_balance: CreditBalance }> {
    const balance = await getCreditBalance(userId);
    
    if (balance.total < amount) {
      return { success: false, new_balance: balance };
    }
    
    // Deduct from subscription credits first (they expire)
    let remaining = amount;
    let subscriptionDeduction = Math.min(remaining, balance.subscription_credits);
    remaining -= subscriptionDeduction;
    
    // Then from purchased credits
    let purchasedDeduction = remaining;
    
    await updateCreditUsage(userId, {
      subscription_credits_used: subscriptionDeduction,
      purchased_credits_used: purchasedDeduction,
    });
    
    const newBalance = await getCreditBalance(userId);
    return { success: true, new_balance: newBalance };
  }
  ```

  ```typescript credit-upsell.ts
      };

      };
    }
  ```

**Pricing psychology:**
- Emergency pack priced high (urgency premium)
- Standard pack shows "best value" badge
- Bulk pack requires sales conversation
</Accordion>

<Accordion title="Rollover strategy">
**Balance retention vs revenue:**

| Strategy | Implementation | Impact |
|----------|----------------|--------|
| **No rollover** | Credits expire monthly | Higher churn, more waste |
| **Full rollover** | Unlimited accumulation | Revenue delay, support complexity |
| **Capped rollover** | Max 2 months worth | Best of both (HeyGen approach) |
| **Paid-only rollover** | Purchased credits never expire | Incentivizes top-ups |

**HeyGen's approach:**
- Subscription credits: Roll over once (900 → 1,800 max)
- Purchased credits: Never expire
- Result: 35% of users buy top-ups, low churn
</Accordion>

## Choosing the Right Model: Decision Matrix

| Model | Best For | Monthly Revenue | Conversion Rate | Implementation Complexity | Churn Risk |
|-------|----------|----------------|----------------|--------------------------|-----------|
| **Credit-Based** | Generation tools, variable usage | $15-95/user | 8-15% | Medium | Medium |
| **Subscription + Limits** | Avatar/editing tools, predictable use | $29-149/user | 5-8% (free→paid) | Medium | Low |
| **Freemium/Watermark** | Editing tools, sunk cost potential | $9-24/user | 25-34% (highest!) | Low | Medium |
| **Tiered Subscriptions** | Diverse personas, feature differentiation | $29-89/user | 7-12% | Medium | Low |
| **Unlimited** | High-volume users, anxiety reduction | $30-50/user | 10-15% | Low | Low |
| **Usage-Based** | APIs, enterprise, unpredictable volume | $500+/user | 15-25% (SMB→Enterprise) | High | Very Low |
| **Hybrid** | Mixed behavior, upsell opportunities | $149+/user | 6-10% + 35% top-up rate | High | Low |

### Selection Guide

<AccordionGroup>
  <Accordion title="Choose Credit-Based if...">
    - **Your compute costs vary wildly** (5s vs 30s video = 6x cost difference)
    - **Users have unpredictable usage** (hobbyist makes 5 videos/month vs creator makes 50)
    - **Quality tiers exist** (draft vs cinematic costs 5x more)
    - **You want volume discounts** ($95 unlimited vs $15 starter = clear upgrade path)
    
    **Examples:** Runway ML, Pika AI, Midjourney
  </Accordion>
  
  <Accordion title="Choose Subscription + Limits if...">
    - **Usage is somewhat predictable** (most users make 10-30min of content/month)
    - **Feature gating is valuable** (custom avatars, voice cloning, API access)
    - **You want stable monthly revenue** (easier forecasting than credit volatility)
    - **Clear personas exist** (solo creator vs agency vs enterprise)
    
    **Examples:** Synthesia, Descript, InVideo
  </Accordion>
  
  <Accordion title="Choose Freemium/Watermark if...">
    - **Users invest time before exporting** (editing = high sunk cost)
    - **Your market is price-sensitive** (social media creators, students)
    - **Watermarks don't kill sharing** (actually helps with virality)
    - **You can offer one-time option** (impulse purchases convert 3x better)
    
    **Examples:** VEED.io, Kapwing, CapCut
  </Accordion>
  
  <Accordion title="Choose Tiered Subscriptions if...">
    - **Distinct customer segments** (individual/pro/team/enterprise with 10x value gaps)
    - **Feature bundling makes sense** (custom avatar = killer feature for Creator tier)
    - **Annual billing is viable** (B2B customers expect annual contracts)
    - **Upsell path is clear** (most users start Starter → upgrade to Creator)
    
    **Examples:** Synthesia, HeyGen, Runway (hybrid with credits)
  </Accordion>
  
  <Accordion title="Choose Unlimited if...">
    - **Usage follows predictable distribution** (90% under 1,000/month)
    - **Users fear metering** ("unlimited" = powerful marketing message)
    - **You can enforce fair use** (soft caps, quality throttling, queue priority)
    - **Costs are manageable** (average user costs you $8, charge $30 = 73% margin)
    
    **Examples:** Luma AI, Canva Pro (for general features)
  </Accordion>
  
  <Accordion title="Choose Usage-Based if...">
    - **Enterprise/API customers** (need predictable per-unit pricing)
    - **Volume is unpredictable** (small business at 100/month vs agency at 50,000/month)
    - **Transparent billing is critical** (enterprises audit every line item)
    - **You want expansion revenue** (usage grows = revenue grows automatically)
    
    **Examples:** Enterprise video APIs, platform integrations
  </Accordion>
  
  <Accordion title="Choose Hybrid if...">
    - **Usage is bimodal** (most use 900 credits, some need 3,000+)
    - **You want upsell revenue** (35% buy top-ups = +$50/user/month)
    - **Rollover reduces churn** (unused credits = sunk cost = retention)
    - **Premium positioning** (hybrid = "advanced" pricing, justifies higher base cost)
    
    **Examples:** HeyGen Business, Adobe Creative Cloud
  </Accordion>
</AccordionGroup>

  </Step>

### Van Westendorp Price Sensitivity Meter

Survey users to find optimal pricing:

**Ask 4 questions:**
1. At what price would this product be **too expensive** (you would not consider it)?
2. At what price would you consider this product **expensive** (but might still buy it)?
3. At what price would you consider this product a **bargain** (a great buy)?
4. At what price would this product be **too cheap** (you'd question the quality)?

**Plot the results:**
- **Point of Marginal Cheapness (PMC):** Where "too cheap" = "expensive"
- **Point of Marginal Expensiveness (PME):** Where "too expensive" = "bargain"
- **Optimal Price Point (OPP):** Where "too expensive" = "too cheap"
- **Indifference Price Point (IPP):** Where "expensive" = "bargain"

**Typical results for AI video tools:**
- PMC: $15-20
- IPP: $29-35 ← **Start here**
- OPP: $45-55
- PME: $75-95
